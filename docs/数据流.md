前端对话时，访问的是 **Java 后端** 的接口，而不是直接访问 Python 服务。整个数据流向如下：

---

## 一、前端 → Java 后端

前端（Vue）调用 **Java 后端** 的接口，例如：

- **发送新消息（对话）**：`POST /api/conversations/{conversationId}/messages`
- 请求体：`{ "content": "用户问题" }`

Java 后端收到请求后，会：
1. 将用户消息存入 MySQL（`conversation_message` 表）。
2. 调用 **Python RAG 服务** 的 `/ask` 接口，获取答案。
3. 将 AI 答案存入 MySQL，并返回给前端。

---

## 二、Java 后端 → Python RAG 服务

Java 后端使用 `RestTemplate` 调用 Python 服务：

```java
// 在 ConversationServiceImpl 中
Map<String, String> requestBody = new HashMap<>();
requestBody.put("question", content);

// 设置超时时间
ClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory();
((SimpleClientHttpRequestFactory) factory).setConnectTimeout(100000);
((SimpleClientHttpRequestFactory) factory).setReadTimeout(100000);
RestTemplate tempRestTemplate = new RestTemplate(factory);

Map<String, Object> ragResponse = tempRestTemplate.postForObject(
        "http://localhost:5001/ask", requestBody, Map.class
);
```

Python 服务返回的 JSON 结构：
```json
{
  "answer": "答案文本",
  "sources": [{"doc_id": 1, "score": 0.95}],
  "thought_process": "思考过程..."
}
```

Java 后端会：
1. 保存 AI 回答到数据库
2. 处理 `sources`，为每个来源添加文档名称
3. 将处理后的结果返回给前端

---

## 三、前端展示

前端收到 Java 后端的响应，包含：
- `answer`：AI 回答
- `sources`：引用的文档列表，每个元素包含：
  - `doc_id`：文档 ID
  - `score`：相似度分数
  - `doc_name`：文档标题（Java 后端添加）
- `thought_process`：思考过程

前端可根据需求展示这些内容（例如用 `white-space: pre-wrap` 展示 `thought_process`）。

## 四、完整接口调用链

```
Vue 前端

   ↓ 检索 → 生成答案
   ↑ 返回 {answer, sources, thought_process}
Java 后端
   ↓ 保存 AI 回答到 MySQL
   ↓ 处理 sources，添加文档标题
   ↑ 返回 {answer, sources, thought_process}
Vue 前端（展示）
```

## 总结

前端只需要调用 **Java 后端** 的 `/api/conversations/{id}/messages` 接口，不需要直接与 Python 服务通信。Java 后端作为统一入口，负责数据库操作和调用 RAG 服务，最终返回整合后的结果。这样前后端职责清晰，也方便后续扩展。
