前端对话时，访问的是 **Java 后端** 的接口，而不是直接访问 Python 服务。整个数据流向如下：

---

## 一、前端 → Java 后端

前端（Vue）调用 **Java 后端** 的接口，例如：

- **发送新消息（对话）**：`POST /api/conversations/{conversationId}/messages`
- 请求体：`{ "content": "用户问题" }`

Java 后端收到请求后，会：
1. 将用户消息存入 MySQL（`conversation_message` 表）。
2. 调用 **Python RAG 服务** 的 `/ask` 接口，获取答案。
3. 将 AI 答案存入 MySQL，并返回给前端。

---

## 二、Java 后端 → Python RAG 服务

Java 后端使用 `RestTemplate` 调用 Python 服务：

```java
// 在 ConversationService 中
AskResponse aiResponse = restTemplate.postForObject(
    "http://localhost:5001/ask",
    new AskRequest(question),
    AskResponse.class
);
```

Python 服务返回的 JSON 结构：
```json
{
  "answer": "答案文本",
  "sources": [{"doc_id": 1, "score": 0.95}],
  "thought_process": "思考过程..."
}
```

Java 再将答案和来源包装后返回给前端。

---

## 三、前端展示

前端收到 Java 后端的响应，包含：
- `answer`：AI 回答
- `sources`：引用的文档 ID 和相似度
- `thought_process`（可选）：思考过程

前端可根据需求展示这些内容（例如用 `white-space: pre-wrap` 展示 `thought_process`）。

---

## 四、完整接口调用链

```
Vue 前端
   ↓ POST /api/conversations/1/messages
Java 后端（Spring Boot）
   ↓ POST http://localhost:5001/ask
Python RAG 服务
   ↓ 检索 → 生成答案
   ↑ 返回 {answer, sources, thought_process}
Java 后端（保存到 MySQL，返回给前端）
   ↑ 返回 {answer, sources, thought_process}
Vue 前端（展示）
```

---

## 五、Java 后端接口示例（供前端调用）

```java
@PostMapping("/api/conversations/{id}/messages")
public ResponseEntity<Map<String, Object>> sendMessage(
        @PathVariable Long id,
        @RequestBody Map<String, String> body) {
    String question = body.get("content");
    // 1. 保存用户消息到 MySQL
    // 2. 调用 Python 服务
    AskResponse aiResp = aiClientService.askQuestion(question);
    // 3. 保存 AI 消息到 MySQL
    // 4. 返回给前端
    Map<String, Object> result = new HashMap<>();
    result.put("answer", aiResp.getAnswer());
    result.put("sources", aiResp.getSources());
    result.put("thought_process", aiResp.getThoughtProcess()); // 可选
    return ResponseEntity.ok(result);
}
```

## 总结

前端只需要调用 **Java 后端** 的 `/api/conversations/{id}/messages` 接口，不需要直接与 Python 服务通信。Java 后端作为统一入口，负责数据库操作和调用 RAG 服务，最终返回整合后的结果。这样前后端职责清晰，也方便后续扩展。
