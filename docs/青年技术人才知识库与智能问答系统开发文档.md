# 青年技术人才知识库与智能问答系统 —— 开发文档

## 1. 项目概述

### 1.1 背景
在企业中，大量宝贵的技术经验、操作规程、故障处理案例以文档形式散落在各处，新员工入职后难以快速获取所需知识，老师傅的经验也无法有效传承。本项目旨在构建一个智能化的知识库系统，利用自然语言处理和大模型技术，实现文档的集中管理、语义检索和智能问答，帮助青年技术人才快速获取所需信息，提升工作效率。

### 1.2 目标
- 实现技术文档的上传、分类、存储和全文检索。
- 支持自然语言提问，系统根据文档内容给出精准答案（基于 RAG 架构）。
- 提供简洁易用的前端界面，包括文档库、搜索、问答交互、热门推荐等。
- 利用本地部署的大模型（如 Qwen-1.8B）保证数据安全，无需依赖外部 API。

## 7. 大模型集成（RAG 实现）

### 7.1 整体流程
1. **文档预处理**：上传文档时，提取纯文本，分割成段落（chunk），为每个段落生成向量，并存储段落文本和向量（内存或 FAISS）。
2. **用户提问**：收到问题后，用同一向量模型将问题转为向量。
3. **相似度检索**：在向量库中找出最相似的 top-k 文档段落。
4. **构造提示词**：将检索到的段落拼接成上下文，与问题一起构造 prompt。
5. **大模型生成**：将 prompt 输入本地大模型（Qwen-1.8B），生成答案。
6. **返回结果**：将答案和来源文档 ID 返回给前端。

### 7.2 关键技术实现
- **向量模型**：`sentence-transformers/all-MiniLM-L6-v2`（384维，轻量）。
- **大模型**：`Qwen/Qwen-1.8B-Chat`（中文效果好，适合 CPU 推理）。
- **向量存储**：简单期间用 Python 列表存储，生产环境可换用 FAISS。
- **文本分割**：按长度（如 500 字符）或段落分割，避免超出模型上下文长度。

### 7.3 代码示例（Python 模块）

```python
# ai/rag_engine.py
from sentence_transformers import SentenceTransformer
from transformers import AutoTokenizer, AutoModelForCausalLM
import numpy as np
import torch

class RAGEngine:
    def __init__(self):
        # 加载向量模型
        self.embed_model = SentenceTransformer('all-MiniLM-L6-v2')
        # 加载大模型
        model_name = "Qwen/Qwen-1.8B-Chat"
        self.tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
        self.llm = AutoModelForCausalLM.from_pretrained(
            model_name,
            trust_remote_code=True,
            torch_dtype=torch.float32,
            device_map="cpu"
        )
        # 存储文档片段
        self.chunk_texts = []      # 文本列表
        self.chunk_doc_ids = []    # 对应的文档ID
        self.chunk_vectors = []    # 向量列表

    def add_document(self, doc_id, content):
        """将文档分割成块并添加到索引"""
        # 简单按长度分割（可优化）
        chunks = self._split_text(content, chunk_size=500)
        for chunk in chunks:
            vec = self.embed_model.encode([chunk])[0]
            self.chunk_texts.append(chunk)
            self.chunk_doc_ids.append(doc_id)
            self.chunk_vectors.append(vec)

    def _split_text(self, text, chunk_size=500, overlap=50):
        """按字符数分割，保留重叠"""
        words = list(text)
        chunks = []
        for i in range(0, len(words), chunk_size - overlap):
            chunk = ''.join(words[i:i+chunk_size])
            chunks.append(chunk)
        return chunks

    def search(self, query, top_k=3):
        """检索最相关的文档片段"""
        query_vec = self.embed_model.encode([query])[0]
        if not self.chunk_vectors:
            return []
        vectors = np.array(self.chunk_vectors)
        sims = np.dot(vectors, query_vec) / (np.linalg.norm(vectors, axis=1) * np.linalg.norm(query_vec))
        top_indices = np.argsort(sims)[-top_k:][::-1]
        results = []
        for idx in top_indices:
            if sims[idx] > 0.5:
                results.append({
                    "doc_id": self.chunk_doc_ids[idx],
                    "text": self.chunk_texts[idx],
                    "score": float(sims[idx])
                })
        return results

    def generate_answer(self, question, context_chunks):
        """根据检索到的片段生成答案"""
        if not context_chunks:
            return "未找到相关文档，请尝试其他问题。"
        context = "\n\n".join([f"[片段{i+1}]{chunk['text']}" for i, chunk in enumerate(context_chunks)])
        prompt = f"""请基于以下文档片段回答用户的问题。如果文档中没有相关信息，请说“文档中没有找到相关内容”。

文档片段：
{context}

问题：{question}

回答："""
        inputs = self.tokenizer(prompt, return_tensors="pt")
        outputs = self.llm.generate(
            inputs.input_ids,
            max_new_tokens=300,
            temperature=0.7,
            do_sample=True,
            pad_token_id=self.tokenizer.eos_token_id
        )
        full_output = self.tokenizer.decode(outputs[0], skip_special_tokens=True)
        answer = full_output[len(prompt):].strip()
        return answer
```

## 10. 注意事项

- **内存占用**：Qwen-1.8B 加载约需 3.6GB 内存，加上其他开销，建议开发机内存 ≥ 8GB。如果不足，换用 0.5B 模型。
- **首次启动**：第一次运行时会从 Hugging Face 下载模型，请确保网络畅通，或配置镜像。
- **向量存储**：示例中使用列表，重启后需重新索引。生产环境建议使用 FAISS 持久化。
- **文本分割**：合理分割能提升检索精度，建议根据文档结构（段落、标题）进行分割。
- **安全性**：上传文件应限制类型和大小，防止恶意文件。

## 11. 扩展思路
