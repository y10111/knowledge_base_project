# 青年技术人才知识库与智能问答系统 —— 开发文档

## 1. 项目概述

### 1.1 背景
在企业中，大量宝贵的技术经验、操作规程、故障处理案例以文档形式散落在各处，新员工入职后难以快速获取所需知识，老师傅的经验也无法有效传承。本项目旨在构建一个智能化的知识库系统，利用自然语言处理和大模型技术，实现文档的集中管理、语义检索和智能问答，帮助青年技术人才快速获取所需信息，提升工作效率。

### 1.2 目标
- 实现技术文档的上传、分类、存储和全文检索。
- 支持自然语言提问，系统根据文档内容给出精准答案（基于 RAG 架构）。
- 提供简洁易用的前端界面，包括文档库、搜索、问答交互、热门推荐等。
- 利用本地部署的大模型（如 Qwen-1.8B）保证数据安全，无需依赖外部 API。

## 3. 系统架构

```
┌─────────────────────┐      ┌─────────────────────────────────┐      ┌─────────────┐
│  前端 Vue2          │────▶│    Java 后端 (Spring Boot)       │────▶│   MySQL     │
└─────────────────────┘      │                                 │      └─────────────┘
                             │  - 文档管理                      │            ▲
                             │  - 对话管理                      │            │
                             │  - 分类管理                      │            │
                             │  - 搜索服务                      │            │
                             └───────────────┬─────────────────┘            │
                                             │                                │
                                             ▼                                │
                                  ┌──────────────────────────┐               │
                                  │  Python RAG 服务 (Flask)  │               │
                                  │ - 向量检索 (BGE-Small-Zh) │               │
                                  │ - 大模型 (Qwen2.5-1.5B)   │               │
                                  │ - FAISS 向量存储          │               │
                                  └──────────────────────────┘               │
```

- **前端**：Vue2 单页应用，通过 Axios 调用 Java 后端 API。
- **Java 后端**：Spring Boot 提供 REST API，集成 Spring Data JPA 操作 MySQL，作为统一接口入口。
- **Python RAG 服务**：独立的 Flask 服务，负责文档向量化、语义检索和答案生成。
- **数据库**：存储文档、对话、消息等数据。
- **向量索引**：使用 FAISS 存储文档向量，实现快速语义检索。
- **大模型**：使用 Qwen2.5-1.5B-Instruct 生成答案。

## 7. 大模型集成（RAG 实现）

### 7.1 整体流程
1. **文档索引**：通过 Python RAG 服务的 `/index` 接口上传文档，服务会提取纯文本，分割成段落（chunk），为每个段落生成向量，并存储到 FAISS 索引中。
2. **用户提问**：前端调用 Java 后端的 `/api/conversations/{id}/messages` 接口。
3. **Java 后端处理**：保存用户消息到数据库，然后调用 Python RAG 服务的 `/ask` 接口。
4. **RAG 服务处理**：
   - 用 BGE 向量模型将问题转为向量
   - 在 FAISS 索引中检索最相似的 top-k 文档段落
   - 将检索到的段落拼接成上下文，构造 prompt
   - 用 Qwen2.5-1.5B-Instruct 大模型生成答案
5. **结果返回**：Python 服务返回答案和来源文档，Java 后端处理后返回给前端。

### 7.2 关键技术实现
- **向量模型**：`BGE-Small-Zh-v1.5`（中文效果好，轻量）。
- **大模型**：`Qwen2.5-1.5B-Instruct`（中文效果好，适合 CPU 推理）。
- **向量存储**：使用 FAISS 进行高效向量检索。
- **文本分割**：按 500 字符分割，保留 50 字符重叠。

### 7.3 Python RAG 服务接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/index` | POST | 文档索引（添加到向量库） |
| `/ask` | POST | 智能问答 |
| `/health` | GET | 健康检查 |

## 8. 开发环境搭建步骤

### 8.1 Java 后端环境
1. **安装 JDK 8**
2. **安装 Maven**
3. **安装 MySQL 8.0**：
   - 可使用 Homebrew：`brew install mysql`
   - 启动：`brew services start mysql`
   - 创建数据库：`CREATE DATABASE knowledge_db CHARACTER SET utf8mb4;`
4. **启动 Java 后端**：
   ```bash
   cd backend
   mvn spring-boot:run
   ```

### 8.2 Python RAG 服务环境
1. **安装 Python 3.10+**（推荐 3.10 或 3.11）
2. **创建虚拟环境**：
   ```bash
   cd python_rag_service
   python3 -m venv venv
   source venv/bin/activate   # Mac
   ```
3. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```
4. **启动 RAG 服务**：
   ```bash
   python app.py
   ```

### 8.3 前端环境（Vue2）
1. **安装 Node.js**（推荐 v16 或 v18）
2. **安装依赖**：
   ```bash
   cd frontend
   npm install
   ```
3. **启动前端**：
   ```bash
   npm run serve
   ```

## 10. 注意事项

- **内存占用**：Qwen2.5-1.5B 加载约需 3GB 内存，加上其他开销，建议开发机内存 ≥ 8GB。
- **首次启动**：第一次运行 Python RAG 服务时会从 Hugging Face 下载模型，请确保网络畅通，或配置镜像。
- **向量存储**：使用 FAISS 进行向量存储，重启服务后需要重新索引文档。
- **文本分割**：系统采用 500 字符分割，保留 50 字符重叠，可根据文档类型调整。
- **安全性**：Java 后端应限制上传文件的类型和大小，防止恶意文件。
- **服务依赖**：Java 后端依赖 Python RAG 服务，确保两个服务都正常运行。
- **数据库配置**：确保 MySQL 数据库已创建，且连接配置正确。
